# calque-blocklist — контроль калек-англицизмов в русскоязычном выводе агента

Инструмент для Claude Code: ведёт межпроектный список запрещённых калек английских техтерминов и во время ответа агента заменяет их на экране на верное русское слово, а на следующей реплике напоминает модели использовать замену впредь.

Калька здесь — англицизм, проникший в русскую прозу агента вместо имеющегося русского слова: транслит («скоуп» вместо «охват»), выдуманный глагол («застейджить»), непереведённое английское слово в русской фразе, псевдотермин или русское слово в кальковом значении («барьер» = gate).

## Из чего состоит

| Часть | Файл | Роль |
|---|---|---|
| Данные | `calque-blocklist.json` | Канонический список: `banned` (калька → замена, вид, основы для ловли) и `accepted` (принятый жаргон). 71 калька, 11 принятых — стартовый набор. |
| Хук распознавания | `hooks/calque-display.py` | MessageDisplay: единственная точка распознавания. Меняет показ на экране, копит находки. |
| Хук подсказки | `hooks/calque-hint-inject.py` | UserPromptSubmit: на следующей реплике отдаёт находки модели как контекст. |
| Скил | `skills/calque-blocklist/` | Ведение списка: запись, валидация, перегенерация стоп-листа, разбор истории. |
| Стоп-лист | `calque-stoplist.md` | Производный короткий список самых частых калек для импорта в `CLAUDE.md`. |

## Как это работает (алгоритм)

Распознавание и подсказка разнесены по двум хукам, чтобы не переделывать уже отправленный ответ, а лишь поправлять показ и подсказывать на будущее.

```
       поток ответа агента                следующая реплика пользователя
                │                                      │
                ▼                                      ▼
   ┌──────────────────────────┐          ┌──────────────────────────────┐
   │  MessageDisplay-хук       │          │  UserPromptSubmit-хук         │
   │  calque-display.py        │          │  calque-hint-inject.py        │
   │                           │          │                               │
   │  оригинал строк ответа    │          │  читает накопленные находки   │
   │  → ловит кальки по detect │          │  → отдаёт модели как контекст │
   │  → substitute: замена на  │ копит    │    (high/low), затем очищает  │
   │    экране + в high        │ ───────▶ │                               │
   │  → hint: оставляет, в low │ pending  │  прошлый ответ НЕ переделыва-  │
   └──────────────────────────┘ по сессии│  ется — учёт впредь           │
                                          └──────────────────────────────┘
```

### Распознавание — `calque-display.py` (MessageDisplay)

Срабатывает на каждую порцию потока, пока ответ агента печатается. По контракту MessageDisplay порция — это целые завершённые строки исходного текста модели, до любой подмены на экране. Это единственное место, которое видит настоящий вывод.

1. Загружает из `calque-blocklist.json` все записи с полем `detect`, компилирует поиск по основам (по началу слова, регистр не важен).
2. Пропускает: блоки кода (` ``` ` / `~~~`), зоны `<meta-discussion>`, код в обратных кавычках, имя инструмента «Анти-калька» и весь ход запуска `/calque-blocklist` (там кальки называются по работе). Распознавание идёт по структуре, а не по угадыванию темы по словам.
3. В обычной прозе:
   - кальку уровня `substitute` заменяет на экране на верное слово. Оформляет Markdown-ссылкой: видимый текст — замена, в URL спрятан оригинал, поэтому пунктирное подчёркивание терминала помечает подмену, а наведение раскрывает, что было заменено. Кладёт в `high`.
   - кальку уровня `hint` на экране оставляет, кладёт в `low`.
4. Меняется только показ. Транскрипт и контекст модели хранят оригинал.
5. Находки копятся по сообщению (`md_<id>.json`); на финальной порции сливаются в очередь по сессии (`<session>.pending.json`).

### Подсказка — `calque-hint-inject.py` (UserPromptSubmit)

На следующей реплике пользователя читает очередь по сессии, отдаёт находки модели как `additionalContext`, сгруппированные по критичности, и очередь удаляет. Прошлый ответ переделывать не нужно — модель учитывает замены в дальнейших формулировках.

### Два уровня критичности (`tier`)

- `substitute` (высокая) — подмена на экране и `high`. По умолчанию для видов `translit`, `coined-verb`, `pseudo-term`.
- `hint` (низкая) — на экране не трогается, идёт в `low` как «проверь по смыслу». По умолчанию для видов `calque-sense`, `untranslated`.

`tier` указывают в записи, только когда уровень отличается от значения по умолчанию для вида.

### Почему распознавание именно в MessageDisplay

Только этот хук видит исходный поток до подмены. Прежний Stop-хук перечитывал транскрипт и ловил не финальный снимок, а промежуточный (показ или поток) — то терял замены, то срабатывал на закавыченные упоминания. Поэтому распознавание перенесено сюда, и оно единственное.

### Контракт данных

```json
{
  "banned": {
    "скоуп": {
      "replace": ["охват", "рамки задачи", "объём"],
      "kind": "translit",
      "note": "scope",
      "detect": ["скоуп"],
      "tier": "substitute"
    }
  },
  "accepted": ["флоу", "коммитить", "пайплайн"]
}
```

- `replace` — обязателен, непустой: замены по контексту.
- `kind` — один из пяти видов: `translit`, `coined-verb`, `untranslated`, `pseudo-term`, `calque-sense`.
- `note` — короткий источник или нюанс.
- `detect` — основы для ловли хуками (без него запись хуками не отслеживается, остаётся только для ручной проверки).
- `tier` — опционально, если уровень отличается от значения по умолчанию для вида.

## Требования

- Claude Code с поддержкой хуков `MessageDisplay` и `UserPromptSubmit`.
- Python 3 (стандартная библиотека, без сторонних пакетов).
- Bash — для `install.sh` и `validate.sh`.

## Установка

### Быстрый путь

```bash
bash install.sh
```

Скрипт копирует данные, хуки и скил в `~/.claude` (или в `$CLAUDE_CONFIG_DIR`, если он задан), делает скрипты исполняемыми, перегенерирует стоп-лист по вашей истории (если она есть) и в конце прогоняет валидацию. Существующий `calque-blocklist.json` не затирается — рядом кладётся `calque-blocklist.json.seed`.

После копирования остаются два ручных шага.

### Шаг 1 — подключить хуки

Добавьте два блока из `settings.hooks.json` в `~/.claude/settings.json` (в секцию `hooks`). Если у вас уже есть хуки `MessageDisplay` или `UserPromptSubmit`, допишите команды в существующие массивы, а не заменяйте их:

```json
{
  "hooks": {
    "MessageDisplay": [
      { "hooks": [ { "type": "command", "command": "python3 ~/.claude/hooks/calque-display.py", "timeout": 10 } ] }
    ],
    "UserPromptSubmit": [
      { "hooks": [ { "type": "command", "command": "python3 ~/.claude/hooks/calque-hint-inject.py", "timeout": 10 } ] }
    ]
  }
}
```

### Шаг 2 — импорт стоп-листа в `CLAUDE.md` (опционально)

Чтобы короткое напоминание висело в каждой сессии, добавьте в свой `~/.claude/CLAUDE.md` строку импорта:

```
@~/.claude/calque-stoplist.md
```

Это не обязательно для работы хуков (они читают `calque-blocklist.json`), но держит частотные кальки перед глазами модели. Раздел-доктрину «Russian Communication Quality» (виды калек, правила перевода) каждый ведёт в своём `CLAUDE.md` сам — он в пакет не входит.

### Ручная установка

Если не хотите запускать скрипт, скопируйте содержимое `claude/` в `~/.claude/` и выполните шаги 1–2:

```bash
cp -r claude/. ~/.claude/
```

## Использование

Скил вызывается как `/calque-blocklist`. Запись в `calque-blocklist.json` — только через скрипты, ручная правка запрещена (иначе теряются удаление дубликатов, сортировка и инварианты).

### Запретить кальку

```bash
python3 ~/.claude/skills/calque-blocklist/scripts/merge_entry.py --json '{
  "тайтл": { "replace": ["заголовок"], "kind": "translit", "note": "title", "detect": ["тайтл"] }
}'
```

Переписать список замен существующей кальки целиком — `replace_set` вместо `replace`.

### Принятый жаргон (никогда не запрещается)

```bash
python3 ~/.claude/skills/calque-blocklist/scripts/merge_entry.py --accepted "флоу,коммитить"
```

### Валидация

```bash
bash ~/.claude/skills/calque-blocklist/scripts/validate.sh
```

Проверяет JSON и инварианты (замены непусты, вид допустим, нет пересечения banned/accepted, ключи отсортированы).

### Перегенерация стоп-листа

```bash
python3 ~/.claude/skills/calque-blocklist/scripts/render_stoplist.py
```

Ранжирует кальки по частоте в собственной прозе агента (по транскриптам сессий) и переписывает `calque-stoplist.md`. Производный файл — руками не править.

### Контекст для верного перевода

```bash
python3 ~/.claude/skills/calque-blocklist/scripts/find_calque_uses.py <слово>
```

Поднимает, как сам агент употреблял термин (группирует по словоформе, печатает свежие цитаты). Помогает перевести верно и проверить, не зацепит ли основа `detect` легальное слово. Вердикта не выносит.

### Пере-наполнение по истории

```bash
python3 ~/.claude/skills/calque-blocklist/scripts/mine_calques.py > /tmp/calque-mining.txt
```

Извлекает окна вокруг маркеров коррекций («кальк», «англицизм», «транслит», стрелки `→`) по всей истории сессий. Результат курируется вручную.

## Состав репозитория

```
calque-blocklist/
├── README.md                  — этот файл
├── install.sh                 — установщик (копирует claude/ в ~/.claude)
├── settings.hooks.json        — блоки хуков для settings.json
├── claude/                    — полезная нагрузка, ложится в ~/.claude
│   ├── calque-blocklist.json  — список (стартовый набор)
│   ├── calque-stoplist.md     — производный стоп-лист
│   ├── hooks/                 — два хука
│   └── skills/calque-blocklist/  — скил и скрипты
└── docs/                      — материалы разработки (анализ и черновой улов калек)
```

## Границы

- Охват — только общие межпроектные кальки. Доменные терминорешения конкретного проекта живут в его файле и сюда не дублируются.
- Подмена касается только показа на экране — транскрипт и контекст модели всегда хранят оригинал.
- Без поля `detect` запись хуками не отслеживается.
- Состояние сессий (`calque_guard/`) — рабочие файлы, в репозиторий не входят.
